한국어로 답해줘야해. 간단한 ERP 웹 앱을 만들거야. 
FASTAPI와 REACT로 Vite는 쓰지 않아. Tailwind는 쓸게. 
일년에 최대 1000건 정도의 주문을 관리할거야. 관리할 업체는 500개 정도. 
필요한 기능은 업체 관리, 제품 관리, 견적 관리, 주문 관리, 생산 관리, 입금 관리, 배송 관리야. 
모든 기능들에는 당연히 CRUD가 적용되어야 하고, 엑셀 파일로 데이터를 업로드/다운로드 할 수 있어야 해. 
10명의 사용자가 있을거야. 모두 같은 권한을 가질거야.
실시간 알림 같은건 필요 없지만, 견적서 발송, 주문 접수 확인서 발송, 배송 시작 알림 이메일 발송 기능은 있어야 해. 이메일은 imap이 아니라 pop3으로 발송 할 수 있어야 해.

업체를 관리할때는 다음 내용들이 포함되어야 해.
- 업체명
- 국가
- 주소
- 담당자명
- 연락처
- export_license_required: yes/no
- export_license_type: 해당 없음/개별 허가/포괄 허가
- export_licnse_number: (포괄 허가일 경우에만 입력됨) string
- shipping_method: string
- shipping_account: string
- 사업자 등록증 및 통장사본과 같은 해당 업체 관련 파일을 업로드 할 수 있어야 해.
엑셀로 다량의 업체를 업로드 하고 다운로드 가능해야해.

제품 관리에는 다음 내용들이 포함되어야 해.
- 품목명: string
- 모델명: string
- 수량: int
- 단가: decimal
- 제조사: string
- description: text
엑셀로 다량의 제품을 업로드 하고 다운로드 가능해야해.

열화상 카메라 모듈은 단독으로 판매 될 수 도 있지만, 열화상 카메라 모듈+렌즈 혹은 열화상 카메라 모듈+렌즈+온도 조합으로 판매 될 수 있어. 
견적과 주문은 연결될 수도 있지만 독립적으로도 존재 가능해.
견적/주문은 한개의 업체에 속하고 여러개의 제품이 포함 될 수 있어. 그리고 견적은 Estimate가 아니라 Quotation이라는 단어로 표기해줘.

--판매--
견적을 제외한 나머지 판매 섹션에 있는 것들은 접수 번호로 관리가 되어야 해.


견적 관리에는 다음 내용들이 포함되어야 해
- 견적일: date
- 업체명
- total_price: decimal
당연히 여러 아이템들이 하나의 견적에 들어갈 수 있어야해.
pdf로 견적서 생성 및 다운로드가 가능해야 하고, 이메일로 발송 할 수 있어야 해.
엑셀로 다량의 견적을 업로드 하고 다운로드 가능해야해.

주문 관리도 견적 관리와 비슷해. 
다만 아래 내용들이 추가적으로 들어가야 해.
- 발주서 번호(PO_number): string
- 접수 번호: string
여러 아이템들이 하나의 주문에 들어갈 수 있어야 해.
해당 주문의 발주서를 업로드 할 수 있어야 해.
pdf로 주문 확인서 생성 및 다운로드가 가능하고, 이메일로 발송 할 수 있어야 해.
엑셀로 다량의 주문을 업로드 하고 다운로드 가능해야해.
주문이 접수되면 해당되는 아이템의 숫자만큼 제품 관리에서 수량이 줄어들어야 해.

생산 관리: 
열화상 카메라 모듈의 시리얼 번호를 입력 할 수 있어야 해.
(추후 시리얼 번호를 입력하는 방법은 바코드 리더기로 입력 할 수 있도록 확장 가능성을 남겨둬야 해)
출고 예정일을 입력 할 수 있어야 해.
pdf로 출고 요청서를 생성 및 다운로드 가능하고, 이메일로 발송 할 수 있어야 해.



입금 관리: 
업체명, 입금일, 입금액을 입력 할 수 있어야 해.
주문 관리의 주문과 연결 될 수 있어야 하는데, 연결되지 않을 수도 있어.
해당 입금 건 관련해서 입금증을 업로드 할 수 있어야 해.
엑셀로 다량의 입금을 업로드 하고 다운로드 가능해야해.

배송 관리:
- 발송일: date
- 접수 번호: string
- 운송장번호: string
- 기타 메모(text)
해당 배송 건 관련해서 운송장과 세금계산서, 수출신고필증을 업로드 할 수 있어야 해.
CI/PL 과 같은 운송 관련 서류 pdf 생성이 가능해야해.
 
--구매--

구매 관리
- 업체명
- 발주일
- 품목명
- 모델명
- 수량
- 단가
- 입고 예정일
- 입고일
- 입금일
- 입금액
해당 배송 건 관련해서 송금증, 운송서류, 세금계산서, 수입신고필증을 업로드 할 수 있어야 해.
CI/PL 과 같은 운송 관련 서류 pdf 생성이 가능해야해.
입고가 완료되면 해당 제품의 수량이 제품 관리에 있는 수량이 추가되어야 해.
