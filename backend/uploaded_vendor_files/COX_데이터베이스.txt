하나의 주문 정보에 여러개의 품목명, 모델명, 옵션, 수량, 단가, 금액이 포함될 수 있어야 함.

Order
- 발주일: 오늘 날짜 자동 입력, yyyy-mm-dd 형식 (수동으로 변경 가능해야 함)
- 발주번호: 직접 입력
- 주문 접수일: 오늘 날짜 자동 입력, yyyy-mm-dd 형식 (수동으로 변경 가능해야 함)
- 접수 번호: 다음과 같은 형식으로 자동 생성:
  •CompanyDB에서 country가 대한민국 의 경우: KR + 주문접수일의 yymm + 해당월의 몇번째 주문인지
    ex) CompanyDB에서 country가 대한민국이고, 2025-03-10에 접수되었으면서, 3번째 주문일 경우: KR2503-03
  •CompanyDB에서 country가 대한민국이 아닐 경우: EX 주문접수일의 yymm + 해당월의 몇번째 주문인지
    ex) 업체명의 국가가 영국이고, 2025-03-10에 접수되었으면서, 4번째 주문일 경우: EX2503-04
- 업체명: CompanyDB의 name을 드롭다운으로 선택 할 수 있어야 함
- 업체별 특별 요청: CompanyDB에서 해당 업체의 업체별 특별 요청을 자동으로 불러와야 함, 수동으로 수정이 가능해야 함.
- 수출 허가 필요 여부: CompanyDB에서 해당 업체의 수출 허가 필요 여부를 자동으로 불러와야 함
- 수출 허가 구분: CompanyDB에서 해당 업체의 수출 허가 구분을 자동으로 불러와야 함
- 수출 허가 번호: CompanyDB에서 해당 업체의 수출 허가 번호를 자동으로 불러와야 함, 수동으로 수정이 가능해야 함.


OrderItem
- 품목명: ProductDB의 category를 드롭다운으로 선택 할 수 있어야 함. (중복 제거)
- 모델명: ProductDB에서 category가 일치하는 model 을 드롭다운으로 선택 할 수 있어야 함.
- 옵션: 직접 입력
- 수량: 직접 입력
- 단가: 직접 입력
- 통화: CompanyDB에서 country가 대한민국 일 경우: KRW 자동 입력 (수동으로 변경 가능해야 함)
          CompanyDB에서 country가 대한민국이 아닐 경우: USD 자동 입력(수동으로 변경 가능해야 함)
- 금액: 수량*단가 자동 입력
- 합계: 해당 주문 정보의 금액 합계


4. 생산 정보: 생산요청일, 생산요청번호, 출고 예정일, 업체명, 품목명, 모델명, 옵션, 수량, 업체 특별 요청, 주문 특별 요청
하나의 생산 정보에 여러개의 품목명, 모델명, 옵션, 수량이 포함될 수 있어야 함.

- 생산요청일: 오늘 날짜 자동 입력, yyyy-mm-dd 형식 (수동으로 변경 가능해야 함)
- 출고 예정일: 오늘 날짜로 부터 28일 후 자동 입력, yyyy-mm-dd 형식 (수동으로 변경 가능해야 함)
- 생산요청번호: 접수 번호 중 선택 혹은 직접 입력 할 수 있어야 함
- 업체명: 해당 접수 번호의 업체명 자동 입력
- 품목명: 해당 접수 번호에 해당하는 모든 품목명 자동 입력
- 모델명: 해당 접수 번호에 해당하는 모든 모델명 자동 입력
- 옵션: 해당 접수 번호에 해당하는 모든 옵션 자동 입력
- 수량: 해당 접수 번호에 해당하는 모든 수량 자동 입력
- 업체 특별 요청: 해당 업체의 업체 특별 요청 자동 입력 (수동으로 변경 가능해야 함)
- 주문 특별 요청: 직접 입력

5. 입금 정보: 입금일, 업체명, 입금액, 통화, 접수 번호

- 입금일: 직접 입력, yyyy-mm-dd 형식
- 업체명: 직접 입력하되, 업체 정보의 업체명 중 하나와 일치해야 함. 만약 일치하는 업체명이 없을 경우에는 팝업이 보이고, 가장 비슷한 업체명들 중 선택 할 수 있어야 함. 
- 입금액: 직접 입력
- 통화: 업체명의 국가가 대한민국 일 경우: KRW 자동 입력 (수동으로 변경 가능해야 함)
          업체명의 국가가 대한민국이 아닐 경우: USD 자동 입력(수동으로 변경 가능해야 함)
- 접수 번호: 직접 입력하되, 주문 정보에 등록된 번호와 일치해야 함.

- 발송일: 오늘 날짜 자동 입력, yyyy-mm-dd 형식 (수동으로 변경 가능해야 함)
